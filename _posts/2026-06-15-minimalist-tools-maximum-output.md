The best tools disappear. They get out of your way and let you focus on the work. Here's my minimalist toolkit.

## The Philosophy

Every tool you add carries a cognitive tax. You have to learn it, configure it, update it, and context-switch into it. The fewer tools, the less friction. I aim for the minimum viable toolkit — just enough to get the job done, nothing more.

## Writing

A text editor and Markdown. That's it. No rich text, no complex formatting. Markdown is portable, readable as plain text, and converts to anything. I write everything — blog posts, notes, documentation — in the same format, in the same editor.

## Task Management

A simple text file called `todo.txt`. One task per line. Top priorities at the top. No categories, no tags, no due dates unless absolutely necessary. The complexity of a task management system should be inversely proportional to the complexity of the work itself.

## Coding

VS Code with a minimal set of extensions (syntax highlighting, a linter, Git integration). No AI copilots, no fancy themes, no dashboard widgets. Just code.

## Communication

Email for async, Signal for urgent. No Slack, no Discord, no Teams unless required by a project. Notification badges are off for everything. I check messages on my schedule, not the sender's.

## The Result

With fewer tools, there are fewer places to check, fewer things to maintain, and fewer distractions to fight. The goal isn't to optimize every tool — it's to optimize the time between using them.
